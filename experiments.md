## 2026-07-17 — efficiency-regression analysis + always-on-prompt trim (commit `d64883e`)

**Analysis (three parallel trace agents): the v2 prompt made early levels ~2x more expensive.** Compared the new 9/9 bp35 (`runs/bp35/20260717-160145`) against the submitted 7/9 (`final-runs/bp35/20260714-070049`) on the identical deterministic L1–L7 puzzles. New run took **491 actions vs 252** (+95%). It was NOT plan fragility: mean plan length actually rose (4.36 vs 3.77) and batch size held. The cost was redundant exploration — **+65 ACTION6 clicks (54% of the L1–3 regression)** were exhaustive tile-probing driven by invented "clear-all-X"/"periodic-phase" mechanics; **surprises 7→21 (3.0×)**; **undo 0→17**; **self-resets 1→8**. Root cause: the v2 additions were **always-on** — the "When a level resists" anti-pruning lines ("'No effect here' never generalizes", "inventory the gaps") and the proactive undo probe, read on every call, taxed every easy level. The escalation ladder was correctly ruled out (first fired at level 9 here; only fires ≥2 self-resets or ≥300 actions). Caveat: `image_prime` was off in the old run, on in the new — a confound for L1 only (priming touches only the first invocation; can't explain L2–L7).

**Fix (commit `d64883e`):** removed the always-on anti-pruning/completionist lines from the system prompt — that posture now reaches the model ONLY via the `[ESCALATION]` directive, which already contained it (directive untouched). Gated the "When a level resists" opener to genuine resistance, added a bounded-check commit line ("do not exhaustively activate every similar object or tile... revisit a settled control only if the level later resists"), and made the undo probe lazy. 40 prompt/runner tests pass, lint clean. Prompt is now leaner, not longer.
